Problèmes liés à la réinitialisation des mots de passe

Réinitialisation des mots de passe pour les comptes d'utilisateurs

Les administrateurs peuvent réinitialiser les mots de passe oubliés afin que les utilisateurs concernés puissent à nouveau accéder à leur compte. Lorsqu'un mot de passe a été réinitialisé, certains types d'informations, tels que les suivants, ne sont plus accessibles :

  • message électronique crypté avec la clé publique de l'utilisateur ;
  • mots de passe Internet enregistrés sur l'ordinateur ;
  • fichiers cryptés par l'utilisateur. Étant donné que le système a besoin du mot de passe de l'utilisateur pour accéder à la clé de décryptage, il ne peut décrypter les clés principales une fois que le mot de passe a été réinitialisé. Les fichiers cryptés ne sont donc pas accessibles, à moins que la clé principale de l'utilisateur n'ait été sauvegardée.
Dans des environnements de domaines, après avoir réinitialisé un mot de passe, vous devez restaurer les clés privées de l'utilisateur pour lui permettre d'accéder à nouveau à ses fichiers et messages électroniques cryptés. Si vous aviez exporté la clé privée EFS (Encrypting File System) du compte de l'utilisateur, vous pouvez la réimporter vers son compte et récupérer l'accès aux fichiers cryptés. En outre, Windows Server 2003 permet aux administrateurs de récupérer les clés privées à l'aide d'un agent de récupération de clé. Si vous n'aviez pas exporté la clé privée et que vous avez défini un agent de récupération de données (DRA, Data Recovery Agent) avant de crypter les fichiers, vous pouvez avoir à nouveau accès aux fichiers EFS en tant qu'agent de récupération de données.

Remarque : dans un environnement autonome, les utilisateurs peuvent créer un disque de réinitialisation du mot de passe pour pouvoir accéder aux fichiers cryptés en cas de réinitialisation du mot de passe d'un compte d'utilisateur local. Pour plus d'informations, consultez la rubrique « Disques de réinitialisation de mot de passe » dans le Centre d'aide et de support de Windows Server 2003.

Réinitialisation des mots de passe pour les comptes d'ordinateurs

Chaque compte d'ordinateur possède un mot de passe qui doit correspondre à un mot de passe stocké dans le service d'annuaire Active Directory. Si ces deux mots de passe ne correspondent pas, Active Directory ne peut pas authentifier l'ordinateur. Par exemple, en cas de problèmes de réplication Active Directory, l'un des mots de passe peut être modifié tandis que l'autre ne change pas. Un ordinateur peut également être hors connexion pendant une durée prolongée. Il est alors possible que le mot de passe Active Directory ait été modifié pendant que l'ordinateur était hors connexion et que le mot de passe local n'ait pas changé. Dans ces deux cas, Active Directory ne sera pas en mesure d'authentifier l'ordinateur. L'utilisateur ne pourra donc pas ouvrir de session sur le réseau. L'administrateur doit alors réinitialiser le mot de passe du compte d'ordinateur.
La réinitialisation du mot de passe restaure les valeurs par défaut du mot de passe stocké sur l'ordinateur et du mot de passe stocké dans Active Directory, permettant ainsi l'authentification de l'ordinateur. Avant de réinitialiser un compte d'ordinateur, tenez compte des restrictions suivantes :

  • Vous devez être membre du groupe Opérateurs de compte, Admins du domaine ou Administrateurs de l'entreprise de Active Directory ou disposer de l'autorisation appropriée. Par mesure de sécurité, utilisez la commande Exécuter en tant que pour effectuer cette procédure.
  • Une fois le compte d'ordinateur réinitialisé, la connexion de l'ordinateur au domaine est rompue. Vous devez donc la rétablir.
